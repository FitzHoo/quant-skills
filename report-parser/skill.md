## 🔧 解析流程

### 步骤1：读取PDF研报
使用Read工具直接读取PDF文件，支持多页读取。

### 步骤2：提取关键信息

**必须提取**：
- ✅ 标题、来源、分析师
- ✅ 策略类型、风险等级
- ✅ 核心观点和投资逻辑
- ✅ **数据源与字段说明**（复现关键）
- ✅ **因子计算步骤（详细）**
- ✅ **计算公式和参数**
- ✅ **代码实现示例**
- ✅ 回测配置与参数
- ✅ 回测表现数据（表格化）
- ✅ 参数优化分析

**可选提取**：
- 投资建议
- 风险提示
- 相关研究引用

### 步骤3：生成标题简称

标题简称用于目录命名，规则：
- 提取核心关键词（2-4个词）
- 使用下划线分隔
- 示例：`扎堆效应_股东户数变动`、`形态识别_均线收敛发散`

### 步骤4：按模板格式输出

将解析结果保存到：
```
02_解析输出/{标题简称}/{标题简称}.md
```

## 🔧 PDF 解析策略（效率优化）

### 并行读取流程

针对大型 PDF（>30页），采用并行读取策略提升效率。

**核心原理**: 在单条 message 中同时发起多个 Read 调用，所有调用并行执行，总耗时≈单次调用耗时。

#### Step 1: 预估 PDF 结构

根据研报类型预估章节位置：

```
典型金融工程研报结构：
- 页1-5:   封面、目录、核心观点
- 页6-15:  文献综述、背景介绍（可跳过）
- 页16-30: 因子/信号构建（核心）
- 页31-45: 回测表现（核心）
- 页46-50: 参数优化
- 页51+:   附录、风险提示
```

#### Step 2: 并行发起 Read 调用

```markdown
# 大型PDF（>30页）并行读取示例
# 单个message中发起多个Read调用

核心观点: Read(pdf, pages="1-5")
因子构建: Read(pdf, pages="16-30")  
回测表现: Read(pdf, pages="31-45")
参数优化: Read(pdf, pages="46-50")（可选）
```

**关键规则**:
- 每批最多 20 页（Read 工具限制）
- 建议每批 10-15 页，确保章节内容完整
- 避免页范围重叠导致重复内容

#### Step 3: 合并处理

```
# 合并所有并行读取结果
# 按模板格式提取关键信息
# 生成结构化输出
```

### PDF 类型处理策略

| PDF类型 | 页数范围 | 读取方案 | 预估耗时 |
|---------|----------|----------|----------|
| 小型 | <30页 | 单次读取全文 | ~5秒 |
| 中型 | 30-60页 | 2-3次并行调用 | ~5秒 |
| 大型 | >60页 | 4-5次并行调用 | ~5秒 |

**加速效果**: 并行读取可节省约 60-70% 的 API 等待时间。

### 智能跳过规则

| 页面类型 | 识别特征 | 处理方式 |
|----------|----------|----------|
| 封面/版权页 | 页码1-2，含"版权所有" | 跳过 |
| 目录页 | 含"目录"字样 | 仅提取结构 |
| 广告宣传页 | 券商宣传内容 | 跳过 |
| 文献综述页 | 大量引用标记 | 跳过或简读 |
| 附录数据页 | 纯数据表格 | 按需提取关键数值 |

### 字段验证规范（新增）

| 验证项 | 要求 | 处理方式 |
|--------|------|----------|
| **表名验证** | 确认表存在于数据库或字典 | 不存在则标注 `[WARN]` |
| **字段名验证** | 确认字段存在于表中 | 不存在则建议替代字段 |
| **中文名匹配** | 字段中文名与字典一致 | 不一致需备注说明 |
| **不确定字段** | 必须查阅数据字典 | 使用 `suggest_field()` 获取建议 |

suggestions = suggest_field('收盘价')
